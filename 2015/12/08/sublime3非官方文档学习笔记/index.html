Sublime Text3非官方文档学习笔记

发表于 2015-12-08   |   分类于 Sublime   |  

Sublime是一款非常好用的代码编辑器，支持Windows、OS X和Linux。目前稳定的版本时2.0.2，Sublime Text3还在测试当中。可以直接从官网下载对应的版本，推荐Sublime Text3，新的版本提供的功能更多。对于一个操作系统，官方提供full version和portable version两个版本，其中full version是普通的安装程序，portable version是免安装版本，下载之后解压可直接运行。我下载的是full version，下载后直接双击安装，完成后打开即可。Sublime需要激活，支持正版可以上官网购买，当然也是可以百度到注册码的。

由于我用的是Windows的操作系统，所以以下内容都是针对Windows系统而言的，在其他系统下快捷键等可能有所不同，但是总体来说差别不大。

基本概念

数据目录：数据目录存放与sublime功能有关的数据，在不同操作系统下会有所不同，在Windows目录下，full version对应的数据目录位于C:\Users\Administrator\AppData\Roaming\Sublime Text 3，portable版本的数据目录在解压目录的Data/文件夹下。如果你找不到自己的数据目录，可以打开sublime，点击工具栏的Preferences->Browse Packages, 打开的目录向上返回一级即为数据目录。

Packages目录：这个目录位于数据目录下的package文件夹，存放Sublime的功能插件和一些依赖包，非常重要。

用户目录：这个文件夹里面保存用户的配置文件，位于packages目录下的User文件夹，是用户的私人目录。

可编程：Sublime不仅仅是一款代码编辑器，而且这款软件本身也是可编程了，它内嵌了一个Python解析器，并且提供了一些API。点击工具栏的views->Show Console或者直接按ctrl + `可以打开控制台，如输入sublime.version()可以查看软件版本号。要退出控制台，直接按ESC键。

插件、包的概念：Sublime强大的一个地方就在于它丰富的插件，这些插件可以让代码编辑更加便捷，如可以自动补全，自动查错，自动排版等等。这些插件都作为一个包存放在packages目录中。

编辑

选中一行： Ctrl + L

交换两个字母的顺序： Ctrl + T

选中

如果需要回退选中操作，可以用ctrl + u, u即undo的缩写。
注： sublime中有两个撤销快捷键,ctrl + u 和 ctrl + z，区别在于，ctrl + z只会记录对代码的改动，而ctrl + u还会记录光标的选择

针对多列的选中

效果： 同时选中多列中的某一个区域，总体选中一个矩形区域，可以同时对这个区域进行操作，就像对一行代码进行操作一样
鼠标操作： 按住shift + 鼠标右键选定，如果需要多区域选定，则选定第二个区域时，按住ctrl + shift + 鼠标右键
键盘操作： ctrl + alt + up 或者ctrl + alt + down

针对单词的选中

鼠标操作： 双击选中当前这一个单词

键盘操作：

  • ctrl + D选中当前单词，再次按下ctrl + D选择下一个单词，
  • ctrl + K表示跳过这个单词，
  • alt + f3则可以选择所有的这个单词，包括其他单词中包含的这个单词的部分

针对多行的选中

鼠标操作： 直接选中多行，如果分隔的，按住ctrl + 鼠标左键拖动

键盘操作：
ctrl + shift + left和ctrl + shift + right左右选中一行中的某个单词，一行选满后回跳到下一行

其他的键盘操作

  • ctrl + up和ctrl + down上下滚动
  • ctrl + shift + up和ctrl + shift + down对选中的内容所在行上下移动

搜索和替换

Sublime提供两种重要形式的搜索，一种是单文件搜索，另一种是多文件搜索，两种搜索都支持正则表达式。

要启用单文件搜索，直接按Ctrl + F，如果要启用正则表达式，必须激活正则表达式选项，位于搜索输入框最左侧的按钮点亮。

文件导航和文件管理

搜索文件

要在项目中搜索某一个文件，可以使用ctrl+p打开搜索框，搜索结果将会显示路径和文件名与输入文本相符的所有文件，使用up和down箭头按钮可以选定某个文件，回车表示打开这个文件。要取消当前搜索，按ESC键。

高级用法：

  1. 同名的文件可能会有多个，如搜索image.js可能会显示多个文件，需要移动光标去选择自己需要的文件。一种简便的方法是，如果你记得文件的路径，可以在搜索时在文件名之前加上文件的路径，如ct/image.js便会很好的过滤搜索结果。文件路径一般不需要太长，一是太长了输入很慢，反而降低了效率，二是长了也根本记不住，一般网上写一级就可以了。
  2. 搜索文件同时定位文件中具体的位置：输入文件名后，看光标选中的是自己需要的文件后，就可以在输入框加入其它搜索条件对文件中的代码做进一步搜索。如：使用@后输入标识符可以搜索指定的函数或者类名，使用#加标识符可以搜索代码中包含搜索内容的单词并且高亮显示，使用：加数字可以定位到具体的行，回车后与搜索相符合的区域就会显示在屏幕上。
  3. 第二项中提到的功能，不仅仅是在搜索文件时，搜索文件打开之后也可以使用。如使用ctrl + R对应输入@的功能，使用ctrl + ;对应输入#的功能，使用ctrl + G 对应输入：的功能。如果不记得这些快捷键，也可以直接输入ctrl + p然后直接输入@、#、：也可以。这种搜索方式与ctrl + F的不同之处在于搜索更加有针对性，而ctrl + F是搜索文件中的所有内容。

侧边栏

侧边栏显示当前项目的所有文件，如果是已有的项目，直接将项目文件夹拖到sublime中打开，如果是从sublime开始建立的项目，那么自然会显示在侧边栏。ctrl + p搜索的文件也正是对应着侧边栏的文件。侧边栏和显示区域有着很强的对应关系。显示区域点击右键，选择reveal in Side Bar可以查看当前文件在侧边栏的位置，从而可以帮助用户了解目录结构。

一般可以直接鼠标操控侧边栏，如果需要键盘控制，用ctrl + 数字0可以将焦点移动到侧边栏上，上下方向键选择文件，左右方向键收起/展开文件夹，回车键打开选中的文件，此时焦点会自动回到编辑区域。如果要手动将焦点回到编辑区域，可以直接按ESC键。

项目文件和工作空间

对于每一个项目，可以建立一个.sublime-project为后缀的json文件，文件中可以保存针对这个项目的特定配置。同一个项目可能有多个workspace,对每一个workspace可以建立一个.sublime-workspace为后缀的文件，保存对这个工作空间的特定配置。

栅格视图

编辑区域在默认的时候通常只能显示一个文件，如果屏幕比较大，那么可以同时显示多个文件。调整方法是，用鼠标点击views->layout选择需要的布局，或者用Alt + shift加数字1~9调整。需要注意的是，键盘上的小键盘区域的数字按键对快捷键不起作用，只有键盘主体区域的数字才起作用。

个性化

用户可以根据自己的喜好配置Sublime，用户的.sublime-settings文件保存在用户目录下，用户目录的路径在基本概念部分已经指出，其他路径中也有一些默认的配置文件。在用户目录下的配置文件在升级时是不会被覆盖的，可以持久化。配置文件是一个JSON文件，用户目录下可以有多个配置文件，文件名可以由用户自己命名，但必须体现出其意义。修改一个配置文件，其效果就会马上显示出来，但是如果有新增的配置文件，必须重新启动软件才能生效。Preferences.sublime-settings是全局的配置文件，对所有类型文件都会起作用，用户可以针对某一种类型文件进行个性化定制，如Python (Linux).sublime-settings，括号中的内容表示针对不同系统的配置，如果没有则是不区分系统的。注意，用户目录下所有针对不同系统的配置文件都会被自动忽略，这样做的原因是用户只用在这里修改一个文件就可以确保所有平台都会被覆盖。

点击Preference->settings - Default可以看到默认的配置文件，在用户目录下的配置文件优先级是最高的，会覆盖默认的配置文件。当然，有些配置值系统会自动调整，也就是用户自己的配置可能不起作用。在控制台中输入view.settings().get("setting_name")可以查看特定属性的当前配置。

配置文件的加载顺序

  • Packages/Default/Preferences.sublime-settings
  • Packages/Default/Preferences (Windows).sublime-settings
  • Packages/AnyOtherPackage/Preferences.sublime-settings
  • Packages/AnyOtherPackage/Preferences (Windows).sublime-settings
  • Packages/User/Preferences.sublime-settings
  • Settings from the current project
  • Packages/Python/Python.sublime-settings
  • Packages/Python/Python (Windows).sublime-settings
  • Packages/User/Python.sublime-settings
  • Session data for the current file
  • Auto-adjusted settings

按照加载顺序，后面的会覆盖前面的，如同CSS文件的加载顺序一样。

快捷键设定

Sublime本身提供了很多快捷键，当然用户也可以自己定义快捷键。快捷键定义的文件也是一个JSON文件，以.sublime-keymap结尾，用户可以在用户文件夹下新建Default.sublime-keymap文件覆盖默认的快捷键设置。

扩展Sublime Text

命令

命令无处不在，包括快捷键、菜单项和宏命令等组成整个命令系统。也可以打开控制台执行命令。

宏命令

宏命令是一个命令集合，它会自动执行一系列的命令，让操作更简单。可以打开tools->Macros查看系统提供的基本宏命令。宏命令的定义在一个以.sublime-macro结尾的JOSN文件中。

如何记录宏命令

如果你需要反复的进行多个相同的操作，那么使用宏命令可以大大简便。按下ctrl + q开始记录，此时可以看到软件做下角有提示，然后一步一步执行操作，完成后点击ctrl + q保存。那么，刚刚执行的所有操作都保存在宏命令缓存区中，如果之前有其他宏命令，会被覆盖，所以这只是一种不持久化的宏命令。如果要使用，直接按ctrl + shift + q就可以自动执行刚刚的宏命令。

如果要自定义可以保存的宏命令，需要在package中创建一个.sublime-macro结尾的JOSN文件，在里面写入多个命令，要执行时直接在tools->Macros里面点击即可。进一步，也可以定义快捷键执行宏文件，从而执行宏命令。

片段

在编写代码时往往有很多需要重复输入的部分，为了避免不需要的重复，可以把这一部分代码写成一个snippet，直接调用插入即可。Sublime本身提供了一些Snippet，可以在tool->Snippets中查看，也可以用户自定义，在用户目录中写入一个以.sublime-snippet结尾的JOSN文件，这个文件可以通过tools->New Snippet创建。

这个JSON文件包括四个部分：

  • content部分：主题内容模版，这是必选的部分
  • tabTrigger: 定义触发按键，输入时点击触发按钮并且按Tab键就可以自动插入这个snippet。如写if时，按tab就可以自动补全，这是Sublime默认提供的。可以把片段理解成代码自动补全的自定义。
  • scope:定义环境范围
  • description：在Snippet菜单中显示时的描述

补全

Sublime自带标识符补全的功能，与上面的片段对代码语法的补全不同，这个补全主要是对已经写过的内容，在后面写到时会自动弹出已经写过的内容选项列表，非常方便。当然，除了系统自带的补全之外，也可以自定义补全的列表。

在列表中选择到自己需要的项后，自动补全有两种方法，按回车键或者Tab键。区别在于：按回车键表示当前的补全已经结束，而按Tab键之后还可以继续按多次，继续显示补全列表中下面的项。如果要按Tab键而不让其实现自动补全功能，那么要按shift + Tab。在自动补全列表中其实也会包含Snippet的功能，因为对函数也会自动补全，并且Snippet补全往往占据在补全列表的第一位。

命令面板

ctrl + shift + p 可以打开命令面板选择对应的命令执行。如果要自定义命令面板，在用户目录下新建一个以.sublime-commands结尾的JSON文件。使用时，用ctrl + shift + p打开，光标选中后回车执行。

语法定义

语法定义就是让Sublime识别编程语言，并且对代码高亮，主要原理是正则表达式匹配，需要安装AAAPackageDev插件，并且写好的以.sublime-syntax为后缀的JSON文件放在用户目录下。

插件

Sublime支持Python插件，目前使用Python 3，熟悉Python的用户可以自己写插件，一般的插件别人都已经开发好了，直接安装使用即可。

包

包是资源的集合，如一个插件就对应一个包。包的存放路径有以下三个：

  • 数据目录文件夹下的Packages文件夹
  • 数据目录文件夹下的Installed Packages文件夹下面的.sublime-package结尾的文件
  • 默认的包文件存放在Application/Packages文件夹下，这个文件夹用户不应该改变。
